    $ dotnet build Algo/Algo.csproj -c Release -p:StockSharpTargets=net6.0 --framework net6.0

Test (the repo's MSTest suite; the new/updated risk tests follow the existing
MSTest + BaseTestClass convention, e.g. `public class PreTradeRiskServiceTests :
BaseTestClass`):

    $ dotnet test Tests/Tests.csproj -c Release --framework net10.0 --nologo

Database note — which tests need SQL and which do not:
  • PURE C# unit/parity tests (rule predicates, boundary math, canonical-limit wiring) run
    with NO database — they are in-memory and deterministic.
  • The live-SQL parity tests (method prefix `Live_*`) exercise the real per-order gate and
    the real position recompute against the same SQL Server described in Database/README.md:
    the Docker container `stocksharp-legacy-sql`, host port 14330. The C# side reads its
    connection from the `STOCKSHARP_LEGACY_SQL_CONNECTION` environment variable, falling back
    (for local disposable runs only) to
    Server=localhost,14330;Database=StockSharpLegacy;User Id=sa;…;TrustServerCertificate=True.
    If that container is not up, the `Live_*` tests are the only ones that need it; the pure
    C# tests still run and pass.
