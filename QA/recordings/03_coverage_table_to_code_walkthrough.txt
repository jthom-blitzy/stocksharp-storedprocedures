### Row 1-2  Price & qty ceilings  (MERGED - duplicative-by-accident)
  canonical : RiskLimitSet.MaxOrderPrice / MaxOrderQty
  gate      : PreTradeRiskService.cs:118  reject "Order price ... meets/exceeds limit ..."
              PreTradeRiskService.cs:122  reject "Order qty ... meets/exceeds limit ..."
  breaker   : Algo/Risk/RiskOrderPriceRule.cs , Algo/Risk/RiskOrderVolumeRule.cs
  Both patterns reject when value >= limit.

### Row 3  Order notional value (qty x price)  (WAS SQL-ONLY -> now a first-class C# rule)
  canonical : RiskLimitSet.MaxOrderValue
  gate      : PreTradeRiskService.cs:129  reject "Order value ... meets/exceeds limit ..."
  breaker   : Algo/Risk/RiskOrderValueRule.cs  (NEW)

### Row 4  Order frequency  (RECONCILED to the stricter rolling-count algorithm)
  canonical : RiskLimitSet.MaxOrderFreqCount / MaxOrderFreqWindowSeconds
  gate      : PreTradeRiskService.cs:131-141  rolling COUNT over "now minus window"
  breaker   : Algo/Risk/RiskOrderFreqRule.cs   rolling-count + late-event watermark
  The single intentional behavioral change: never less strict than the old fixed window.

### Row 5  Resulting position size  (PRESERVED-DISTINCT - shares one threshold)

  gate      : PreTradeRiskService.cs:144-152  checks the HYPOTHETICAL post-fill position
  breaker   : Algo/Risk/RiskPositionSizeRule.cs  checks the CURRENT position
  Same threshold; two evaluation contexts answering different questions (AAP 0.6.2).

### Row 6  Cumulative commission  (PRESERVED-DISTINCT - shares one threshold)

### Row 7  Daily traded volume  (WAS SQL-ONLY -> now a first-class C# rule)
